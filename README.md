-  Question 2: HTML Elements
1. Most Challenging Elements
- **`<table>`**  
  Difficult to maintain proper structure using `thead`, `tbody`, and `tfoot`, especially with `colspan`.
- **`<datalist>`**  
  Required understanding how it differs from `<select>` and linking it correctly using `id` and `list`.
- **`<audio>` and `<source>`**  
  Managing file paths and understanding browser compatibility was challenging.
- **`<meter>` vs `<progress>`**  
  Initially confusing; learned their different purposes.
- **`<fieldset>` and `<legend>`**  
  Required a shift toward accessibility-focused grouping.
2. Use of Semantic Elements
- **`<header>`** – Contains name and navigation  
- **`<nav>`** – Navigation links  
- **`<main>`** – Main page content  
- **`<section>`** – Divides content into logical blocks  
- **`<article>`** – Represents standalone content (projects)  
- **`<aside>`** – Supplementary content  
- **`<footer>`** – Contact info and copyright
-  3. Most Useful Element
**`<section>`**  
It helped organize the layout into clear, logical blocks, improving readability and accessibility without CSS.
 Attributes
1. Essential Attributes
- **`action` & `method` (form)**  
  Define how and where form data is sent.
- **`href` (anchor tags)**  
  Enables navigation and linking.
- **`type` (input)**  
  Improves usability and validation.
 2. `class` vs `id`
- **`id`** – Unique identifier (e.g., linking sections)  
- **`class`** – Reusable grouping for styling or structure
-  3. Best UX Attribute
**`required`**  
Prevents empty form submission and provides instant browser feedback.
